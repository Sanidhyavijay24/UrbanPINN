# 🏙️ UrbanPINN: AI-Driven Microclimate CFD Simulator

UrbanPINN is a production-grade, full-stack application that leverages **Physics-Informed Neural Networks (PINNs)** to predict and visualize real-time 3D fluid dynamics across complex urban geometries. 

By replacing traditional, computationally heavy Navier-Stokes (RANS) solvers with a continuous sinusoidal neural network (SIREN), this application allows users to instantly slice and analyze aerodynamic wind vectors and ambient thermal densities in a full WebGL 3D environment.

> 🧠 **Note:** This repository contains the high-performance **API Inference Engine** and the **Next.js WebGL Dashboard**. For the raw PyTorch neural network architecture, data formulation, and training pipeline, please visit the companion repository: [Sanidhyavijay24/UrbanPINN_Model](https://github.com/Sanidhyavijay24/UrbanPINN_Model)

---

## 🚀 System Architecture

### 1. Neural Inference Backend (FastAPI / PyTorch)
*   Loads a highly compressed (~4.7MB) multiscale **SIREN** model capable of mapping continuous $(x, y, z)$ spatial coordinates to highly constrained $(u, v, w, p, T)$ Navier-Stokes solutions.
*   Serves the generated multiphysics arrays via a high-performance **FastAPI** route structure optimized for frontend polling and telemetry extraction.

### 2. Physical Rendering Engine (Next.js / React-Three-Fiber)
*   **Volumetric 3D Mapping**: Renders massive bounding arrays of the Manhattan city layout utilizing native Trimesh ear-clipping to prevent non-convex geometric artifacts.
*   **Multi-Spectrum HUD**: Dynamically tracks and interpolates "Wind Velocity" and "Ambient Thermal" variations generated by the AI, pushing WebGL gradients natively beneath the physical meshes.
*   **Frosted Refractive Glass**: Utilizes advanced `MeshPhysicalMaterials` to render city structures with high volumetric transmission, giving the system cinematic depth while retaining true depth-sorting to mask internal AI vector anomalies.

## ⚙️ Running Locally

UrbanPINN is designed as a split monolith. You must run the FastAPI backend alongside the Next.js development server to stream the live physics inferences.

### 1. Booting the Neural Engine (Backend)
Navigate to the `backend/` directory, ensure your Anaconda environment is active, and launch the Uvicorn server:
```bash
cd backend
pip install -r requirements.txt
uvicorn api.v1.main:app --reload --port 8000
```
*The endpoint will be available at `http://localhost:8000/api/v1/predict/slice`.*

### 2. Booting the Visualization Dashboard (Frontend)
Open a separate terminal, navigate to the `frontend/` directory, and install dependencies using **Bun**:
```bash
cd frontend
bun install
bun run dev
```

Navigate to `http://localhost:3000` in your Chromium/Webkit browser to interact with the CFD visualization.

## 🎯 Physical Capabilities & Accuracy
The PINN effectively mimics aerodynamic properties natively inside the React scene:
*   **No-Slip Geometry Enclosures**: Wind instances structurally occluded and mapped safely outside city building geometry boundaries.
*   **Thermal Mapping**: Continuous monitoring overlay plotting thermal heat islands inside dense city quarters directly underneath the wind physics. 
*   **Aerodynamic Wake Zones**: Lower-threshold constraints allowing structural wake capture inside densely packed alleyways.
